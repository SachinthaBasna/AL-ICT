Introduction to Number Systems

What are Number Systems?

A number system is a way to represent numbers using a specific set of symbols and rules. Computers use different number systems to store and process data efficiently.

Why Different Number Systems?
Computers work with electricity (ON/OFF states), making binary perfect for digital circuits. Hexadecimal provides a compact way to represent binary data, while octal was historically used in computing.

Decimal Number System (Base 10)

The System We Use Daily

The decimal system uses 10 digits (0-9) and is based on powers of 10.

Example: 2345 in Decimal
2345 = (2 × 10³) + (3 × 10²) + (4 × 10¹) + (5 × 10⁰)
= (2 × 1000) + (3 × 100) + (4 × 10) + (5 × 1)
= 2000 + 300 + 40 + 5
= 2345

Binary Number System (Base 2)

The Language of Computers

Binary uses only 2 digits (0 and 1) representing OFF and ON states in computer circuits.

Why Binary?
Computers use transistors that can be in two states: ON (1) or OFF (0). This makes binary the natural choice for digital systems.

Octal Number System (Base 8)

Base 8 System

Octal uses 8 digits (0-7) and is based on powers of 8. Often used in computing for representing binary in a more compact form.

Octal Positional Values
8³ 8² 8¹ 8⁰
512 64 8 1
7 5 6 3
Example: 7563₈ to Decimal
= (7 × 8³) + (5 × 8²) + (6 × 8¹) + (3 × 8⁰)
= (7 × 512) + (5 × 64) + (6 × 8) + (3 × 1)
= 3584 + 320 + 48 + 3
= 3955₁₀
Octal-Binary Connection:
Each octal digit represents exactly 3 binary digits (bits):
7₈ = 111₂, 5₈ = 101₂, 6₈ = 110₂, 3₈ = 011₂

Number System Conversions

Decimal to Binary (Division Method)

Method: Repeatedly divide by 2 and record remainders (read from bottom to top)

Example: Convert 45₁₀ to Binary
45 ÷ 2 = 22  remainder 1  ←
22 ÷ 2 = 11  remainder 0  ↑
11 ÷ 2 = 5   remainder 1  ↑
5 ÷ 2  = 2   remainder 1  ↑
2 ÷ 2  = 1   remainder 0  ↑
1 ÷ 2  = 0   remainder 1  (Start here)

Answer: 101101₂
Binary to Decimal (Expansion Method)

Method: Multiply each bit by its positional value and add

Example: Convert 101101₂ to Decimal
1×32 + 0×16 + 1×8 + 1×4 + 0×2 + 1×1
= 32 + 0 + 8 + 4 + 0 + 1
= 45₁₀
Binary ↔ Hexadecimal (Grouping Method)

Method: Group binary digits in sets of 4 (from right to left)

Example: Convert 11010111₂ to Hex
Step 1: Group into 4 bits → 1101 0111
Step 2: Convert each group:
        1101₂ = 13₁₀ = D₁₆
        0111₂ = 7₁₀  = 7₁₆

Answer: D7₁₆
Reverse (Hex to Binary): Each hex digit → 4 binary digits
A3₁₆ = 1010 0011₂ (A=1010, 3=0011)

Finger Binary Counting

Count from 0 to 31 on one hand using binary!

How it works:
• Thumb = 1 (2⁰)
• Index = 2 (2¹)
• Middle = 4 (2²)
• Ring = 8 (2³)
• Pinky = 16 (2⁴)

Example: To show 7:
Thumb (1) + Index (2) + Middle (4) = 7

Hex-Binary Quick Conversion

Remember: 1 Hex digit = 4 Binary digits

Quick Check Methods
1. Binary to Decimal (Doubling Method)
For binary 1011₂:
Start from left: (((1×2)+0)×2+1)×2+1 = 11
This is faster for mental calculation!
2. Check if Conversion is Correct
Binary digit count check:
• 1-bit number: 0-1 (2 values)
• 2-bit number: 0-3 (4 values)
• 3-bit number: 0-7 (8 values)
• 4-bit number: 0-15 (16 values)
• 8-bit number: 0-255 (256 values)

A/L Exam Tips
  • Always show working: Write each step clearly for partial marks
  • Use subscript notation: 101₂, 25₁₀, 1F₁₆ to avoid confusion
  • Verify your answer: Convert back to check if you get the original number
  • Common mistakes: • Forgetting to read remainders bottom-to-top
    • Using 8 or 9 in octal (only 0-7 allowed)
    • Confusing O (letter) with 0 (zero) in hex
  • Time management: Binary↔Hex conversions are fastest, do those first!

Question 2: Binary to Hex

Convert 110110101011₂ to hexadecimal

Answer: DAB₁₆

Working:
Group into 4 bits: 1101 1010 1011
• 1101₂ = 13₁₀ = D₁₆
• 1010₂ = 10₁₀ = A₁₆
• 1011₂ = 11₁₀ = B₁₆
Question 3: Hex to Decimal

What is the decimal value of 2F3A₁₆?

Answer: 12090₁₀

Working:
2F3A₁₆ = (2×16³) + (F×16²) + (3×16¹) + (A×16⁰)
= (2×4096) + (15×256) + (3×16) + (10×1)
= 8192 + 3840 + 48 + 10
= 12090₁₀
